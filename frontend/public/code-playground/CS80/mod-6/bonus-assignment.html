Problem 1 – 16.3: Client/Server Interactions – Traditional vs. Ajax

Traditional Web Application

In a traditional web application, when a user clicks a button like “Load Data”, the browser sends a request to the server and reloads the entire page. The server returns a full new HTML response. This process is synchronous and causes a full page refresh, even if only a small part of the content changes.

Ajax Web Application (Example from Lecture)

In an Ajax-based application, clicking a button triggers a JavaScript (jQuery) function that sends a request to the server in the background. Only the needed data is returned (usually in JSON format), and the page updates dynamically without a full reload.

Here’s the example from the lecture notes (CodeSandbox):

$("#btn").click(function () {
  $.ajax({
    method: "GET",
    url: "https://baconipsum.com/api/?type=all-meat",
    dataType: "json"
  })
    .done(function (data) {
      $(".baconText").text(data[0]);
    })
    .fail(function () {
      alert("no good");
    });
});

In this example, the user clicks a button with ID #btn. Instead of reloading the page, jQuery sends a GET request to the Bacon Ipsum API. The server returns JSON, and the first paragraph is inserted into the page using $(".baconText").text(data[0]).

Summary
  • Traditional: Button click → full request to server → entire page reloads with new HTML
  • Ajax: Button click → async request via jQuery → only part of the page updates
