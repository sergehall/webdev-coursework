Executive Summary

AWS Learning Portal is an interactive educational web application designed to translate CS 79C coursework into a live AWS portfolio project. Instead of documenting services in isolation, the application demonstrates how EC2, S3, Lambda, API Gateway, DynamoDB, CloudWatch, and supporting AWS networking services work together in a cohesive system.

The application uses a Next.js frontend and a NestJS backend written in TypeScript. It includes course modules, live API demonstrations, protected user flows, S3 uploads, and DynamoDB-backed activity logging. The repository supports a low-cost single-EC2 demo workflow, and it also configured a static Public IPv4 address, 13.59.11.226, using an Elastic IP for stable public access to the deployed environment.

Architecture and AWS Services

The frontend serves user-facing pages such as Home, Modules, Live AWS, Uploads, Architecture, API Demo, Final Project, and Cabinet. The NestJS backend exposes REST endpoints for modules, uploads, logs, authentication, password recovery, Lambda demos, and student profile management. Amazon S3 stores uploaded files, Lambda powers the serverless logging workflow, API Gateway exposes the Lambda integration, DynamoDB stores activity records, and CloudWatch provides dashboards, alarms, and execution logs.

Implementation and Security

Application Stack

Next.js 16 and React 19 power the frontend, while NestJS 11 and AWS SDK v3 power the backend. PM2 is used for process management in the EC2-based deployment.

Security Controls

The app uses JWT authentication, backend profile validation, optional Cloudflare Turnstile, upload type and size limits, and throttling on uploads plus sensitive authentication routes.

Scalability and Availability

The AWS deployment uses EC2 networking, public subnet access, and an Elastic IP to provide stable connectivity through the public IPv4 address 13.59.11.226 while keeping the infrastructure simple and cost-conscious.

Monitoring

CloudWatch dashboards, CPU alarms, Lambda execution logs, and the `/health` endpoint provide operational visibility for both infrastructure and application behavior.

Testing, Cost, and Cleanup

The repository includes root-level lint, typecheck, smoke-test, unit-test, and e2e-test commands. Manual browser checks were also used for registration, login, uploads, API Demo, Cabinet pages, and the featured Final Project flow. Cost control was part of the design: local development was used whenever possible, small instance sizes were chosen for AWS testing, and a documented resource cleanup sequence is included in the README.

After grading, the main billable resources to remove are the EC2 instance, Elastic IP, S3 bucket contents and bucket, DynamoDB table, Lambda function, API Gateway, CloudWatch resources, IAM roles, and any VPC resources created only for this project.
